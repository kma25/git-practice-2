## Введение

Git — это распределенная система контроля версий. Она позволяет сохранять историю изменений проекта, работать с ветками и возвращаться к предыдущим версиям файлов.

GitHub — это онлайн-сервис для хранения Git-репозиториев и совместной работы над проектами.

---

## Создание учетной записи GitHub

Для создания учетной записи GitHub необходимо:

1. Перейти на сайт GitHub.
2. Нажать кнопку `Sign up`.
3. Ввести адрес электронной почты.
4. Придумать пароль.
5. Указать имя пользователя.
6. Подтвердить регистрацию через письмо на email.
7. После входа в аккаунт создать новый репозиторий.

Пример названия репозитория:

```text
git-instruction
```

## Установка Git

Для установки Git необходимо:

1. Перейти на официальный сайт Git.
2. Скачать установщик для Windows.
3. Запустить установщик.
4. На этапах установки оставить стандартные параметры.
5. После установки открыть Git Bash.

Проверка установки Git:

```bash
git --version
```

Пример результата:

```text
git version 2.45.0
```

## Локальная работа с Git

Локальная работа с Git выполняется на компьютере пользователя. На этом этапе создается репозиторий, добавляются файлы, создаются коммиты и выполняется работа с ветками.

### Инициализация репозитория

Команда создает новый локальный Git-репозиторий в текущей папке.

```bash
git init
```

Пример:

```bash
git init
```

### Проверка состояния репозитория

Команда показывает состояние файлов: измененные, добавленные и неотслеживаемые.

```bash
git status
```

Пример:

```bash
git status
```

### Добавление файла в индекс

Команда добавляет файл в область подготовленных изменений.

```bash
git add имя_файла
```

Пример:

```bash
git add README.md
```

### Добавление всех файлов

Команда добавляет все измененные и новые файлы.

```bash
git add .
```

Пример:

```bash
git add .
```

### Создание коммита

Коммит фиксирует изменения в истории репозитория.

```bash
git commit -m "Сообщение коммита"
```

Пример:

```bash
git commit -m "Добавлена инструкция по локальному Git"
```

### Просмотр истории коммитов

Команда показывает историю изменений.

```bash
git log
```

Пример краткого вывода:

```bash
git log --oneline
```

### Создание ветки

Команда создает новую ветку.

```bash
git branch имя_ветки
```

Пример:

```bash
git branch local
```

### Переключение между ветками

Команда переключает пользователя на выбранную ветку.

```bash
git checkout имя_ветки
```

Пример:

```bash
git checkout local
```

Также можно использовать современную команду:

```bash
git switch local
```

### Создание ветки и переход в нее

Команда создает новую ветку и сразу переключается на нее.

```bash
git checkout -b имя_ветки
```

Пример:

```bash
git checkout -b local
```

### Слияние веток

Команда объединяет изменения из выбранной ветки с текущей веткой.

```bash
git merge имя_ветки
```

Пример:

```bash
git merge local
```

## Работа с удаленным репозиторием

Удаленный репозиторий хранится на GitHub или другом Git-сервисе. Он нужен для публикации проекта, резервного хранения кода и совместной работы.

### Клонирование репозитория

Команда создает локальную копию удаленного репозитория.

```bash
git clone ссылка_на_репозиторий
```

Пример:

```bash
git clone https://github.com/user/git-instruction.git
```

### Добавление удаленного репозитория

Команда связывает локальный репозиторий с удаленным.

```bash
git remote add origin ссылка_на_репозиторий
```

Пример:

```bash
git remote add origin https://github.com/user/git-instruction.git
```

### Просмотр удаленных репозиториев

Команда показывает список подключенных удаленных репозиториев.

```bash
git remote -v
```

Пример:

```bash
git remote -v
```

### Отправка изменений на GitHub

Команда отправляет локальные коммиты в удаленный репозиторий.

```bash
git push origin имя_ветки
```

Пример:

```bash
git push origin main
```

### Первая отправка ветки

Команда отправляет ветку на GitHub и связывает локальную ветку с удаленной.

```bash
git push -u origin имя_ветки
```

Пример:

```bash
git push -u origin global
```

### Получение изменений из удаленного репозитория

Команда скачивает изменения и объединяет их с текущей веткой.

```bash
git pull origin имя_ветки
```

Пример:

```bash
git pull origin main
```

### Загрузка изменений без автоматического слияния

Команда скачивает информацию об изменениях, но не объединяет их автоматически.

```bash
git fetch
```

Пример:

```bash
git fetch origin
```

## Работа с форками

Форк — это копия чужого репозитория в своем аккаунте GitHub. Форки используются, когда нужно предложить изменения в чужой проект без прямого доступа к исходному репозиторию.

### Принцип работы с форком

1. Открыть чужой репозиторий на GitHub.
2. Нажать кнопку `Fork`.
3. GitHub создаст копию репозитория в аккаунте пользователя.
4. Склонировать форк на компьютер.
5. Создать отдельную ветку.
6. Внести изменения.
7. Отправить изменения в свой форк.
8. Создать Pull Request в исходный репозиторий.

### Клонирование форка

Команда создает локальную копию форка.

```bash
git clone ссылка_на_форк
```

Пример:

```bash
git clone https://github.com/user/project-fork.git
```

### Добавление исходного репозитория

Исходный репозиторий обычно добавляется под именем `upstream`.

```bash
git remote add upstream ссылка_на_исходный_репозиторий
```

Пример:

```bash
git remote add upstream https://github.com/original-owner/project.git
```

### Получение изменений из исходного репозитория

Команда загружает изменения из оригинального репозитория.

```bash
git fetch upstream
```

Пример:

```bash
git fetch upstream
```

### Обновление своей ветки из исходного репозитория

Команда объединяет изменения из исходного репозитория с текущей веткой.

```bash
git merge upstream/main
```

Пример:

```bash
git merge upstream/main
```

## Итоговая схема работы

Итоговый вариант инструкции получается после слияния веток `local` и `global` в основную ветку `main`.

```text
main   — структура документа, создание GitHub и установка Git
local  — команды для локальной работы с Git
global — команды для удаленного Git и форков
```

Общая последовательность выполнения практической работы:

```bash
git checkout main
git checkout -b local
git add README.md
git commit -m "Добавлены команды локального Git"
git push -u origin local

git checkout main
git checkout -b global
git add README.md
git commit -m "Добавлены команды удаленного Git и форки"
git push -u origin global

git checkout main
git merge local
git merge global
git push origin main
```
